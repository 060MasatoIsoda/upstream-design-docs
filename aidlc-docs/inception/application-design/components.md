# コンポーネント定義

## 1. フロントエンド層

### 1.1 FE-AUTH: 認証コンポーネント
- **目的**: SAML/OIDC認証フローの処理、トークン管理
- **責務**:
  - IdP（Azure AD / Okta）へのリダイレクトとコールバック処理
  - アクセストークン・リフレッシュトークンの管理
  - 認証状態の保持とセッション管理
  - 未認証ユーザーのルートガード

### 1.2 FE-EXPENSE: 経費精算画面コンポーネント
- **目的**: 経費精算申請の起票・編集・表示
- **責務**:
  - 経費精算申請フォーム（新規作成・編集・下書き保存）
  - 明細行の追加・削除・編集
  - 添付ファイルのアップロード・プレビュー
  - 申請の提出・取消し

### 1.3 FE-APPROVAL: 承認画面コンポーネント
- **目的**: 承認待ち一覧の表示と承認・差戻し操作
- **責務**:
  - 承認待ち申請一覧の表示
  - 申請詳細の表示（添付ファイル含む）
  - 承認・差戻し操作（コメント付与）

### 1.4 FE-INQUIRY: 照会画面コンポーネント
- **目的**: 申請状況の検索・一覧表示
- **責務**:
  - 申請一覧の表示（申請者向け・承認者向け）
  - 検索条件によるフィルタリング
  - 申請詳細への遷移

### 1.5 FE-DASHBOARD: ダッシュボード画面コンポーネント
- **目的**: 管理者向け集計・統計情報の表示
- **責務**:
  - 申請件数・処理状況の集計グラフ表示
  - 部門別・期間別の集計
  - 滞留申請のアラート表示

### 1.6 FE-ADMIN: 管理画面コンポーネント
- **目的**: ユーザー管理・承認ルート設定・監査ログ閲覧
- **責務**:
  - ユーザー一覧・登録・編集・無効化
  - ロール割り当て
  - 承認ルート設定
  - 代理設定
  - 監査ログ検索・閲覧

### 1.7 FE-COMMON: 共通UIコンポーネント
- **目的**: アプリケーション全体で共有するUI部品
- **責務**:
  - レイアウト（ヘッダー・サイドバー・フッター）
  - 通知トースト
  - ローディング表示
  - エラーハンドリング表示
  - 多言語切替（日本語・英語）

## 2. バックエンド層（API）

### 2.1 BE-AUTH: 認証・認可サービス
- **目的**: SAML/OIDC認証の検証、RBAC制御
- **責務**:
  - IdPからのトークン検証
  - ユーザーセッション管理
  - ロールベースアクセス制御（RBAC）
  - APIエンドポイントの認可ミドルウェア

### 2.2 BE-EXPENSE: 経費精算サービス
- **目的**: 経費精算申請のCRUD操作とビジネスロジック
- **責務**:
  - 経費精算申請の作成・更新・取得・削除
  - 明細行の管理
  - 下書き保存
  - 申請の提出・取消し
  - 入力バリデーション

### 2.3 BE-WORKFLOW: 承認ワークフローサービス
- **目的**: 承認ルートの判定と承認プロセスの管理
- **責務**:
  - 承認ルートの自動判定（固定2段階: 直属上長 → 部門長）
  - 承認・差戻し処理
  - 代理承認の処理
  - 申請ステータスの遷移管理

### 2.4 BE-USER: ユーザー管理サービス
- **目的**: ユーザー情報・ロール・組織情報の管理
- **責務**:
  - ユーザーのCRUD操作
  - ロール・権限の管理
  - 所属部門・上長・部門長の管理
  - 代理設定の管理

### 2.5 BE-ROUTE: 承認ルート管理サービス
- **目的**: 承認ルートの設定・管理
- **責務**:
  - 部門ごとの承認ルート設定
  - 承認ルートの有効期間管理
  - 組織改編時のルートメンテナンス

### 2.6 BE-DASHBOARD: ダッシュボードサービス
- **目的**: 集計データの生成・提供
- **責務**:
  - 申請件数・処理状況の集計
  - 部門別・期間別の集計
  - 滞留申請の検出

### 2.7 BE-NOTIFICATION: 通知サービス
- **目的**: メール通知の送信
- **責務**:
  - 通知テンプレートの管理
  - メール送信（Amazon SES経由）
  - 通知履歴の記録

### 2.8 BE-FILE: ファイル管理サービス
- **目的**: 添付ファイルのアップロード・ダウンロード管理
- **責務**:
  - ファイルアップロード（S3への保存）
  - ファイルダウンロード（署名付きURL生成）
  - ファイルサイズ・形式のバリデーション
  - ファイルメタデータの管理

### 2.9 BE-AUDIT: 監査ログサービス
- **目的**: 操作ログ・監査ログの記録と検索
- **責務**:
  - 全操作の監査ログ記録
  - 監査ログの検索・閲覧API
  - ログの保管期間管理

### 2.10 BE-BATCH: バッチ処理サービス
- **目的**: 外部システムとのバッチ連携
- **責務**:
  - 人事マスタ取込（社員情報・所属部門・役職）
  - 組織マスタ取込（組織階層）
  - 会計システム向け仕訳データ出力
  - バッチ実行スケジュール管理
  - バッチ実行結果のログ記録

## 3. データアクセス層

### 3.1 DA-REPO: リポジトリ層
- **目的**: データベースアクセスの抽象化
- **責務**:
  - SQLAlchemy ORMモデルの定義
  - CRUD操作の実装
  - クエリの構築と実行
  - トランザクション管理

### 3.2 DA-MIGRATION: マイグレーション管理
- **目的**: データベーススキーマのバージョン管理
- **責務**:
  - Alembicによるマイグレーションスクリプト管理
  - スキーマの変更履歴管理
