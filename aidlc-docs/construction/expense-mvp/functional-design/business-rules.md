# ビジネスルール定義 — expense-mvp

## 1. 入力バリデーションルール

### 1.1 経費精算申請

| ルールID | フィールド | ルール | エラーメッセージ |
|---------|----------|--------|---------------|
| VR-001 | title | 必須、1〜200文字 | タイトルは必須です（200文字以内） |
| VR-002 | expense_items | 1件以上の明細行が必要 | 明細行を1件以上入力してください |
| VR-003 | total_amount | 1円以上、1,000,000円以下 | 合計金額は1円以上100万円以下にしてください |
| VR-004 | 各明細.amount | 1円以上 | 金額は1円以上を入力してください |
| VR-005 | 各明細.description | 必須、1〜500文字 | 摘要は必須です（500文字以内） |
| VR-006 | 各明細.incurred_date | 必須、過去日のみ | 発生日は過去の日付を入力してください |
| VR-007 | 各明細.expense_type_id | 必須、有効な経費種別 | 経費種別を選択してください |

### 1.2 添付ファイル

| ルールID | ルール | エラーメッセージ |
|---------|--------|---------------|
| VR-010 | 1ファイルあたり5MB以下 | ファイルサイズは5MB以下にしてください |
| VR-011 | 1申請あたり合計10MB以下 | 添付ファイルの合計サイズは10MB以下にしてください |
| VR-012 | 対応形式: PDF, JPEG, PNG | 対応していないファイル形式です（PDF, JPEG, PNGのみ） |

### 1.3 ユーザー管理

| ルールID | フィールド | ルール | エラーメッセージ |
|---------|----------|--------|---------------|
| VR-020 | employee_number | 必須、一意、1〜20文字 | 社員番号は必須です（一意） |
| VR-021 | email | 必須、一意、メール形式 | 有効なメールアドレスを入力してください |
| VR-022 | name_ja | 必須、1〜100文字 | 氏名（日本語）は必須です |
| VR-023 | department_id | 必須、有効な部門 | 所属部門を選択してください |

### 1.4 代理設定

| ルールID | ルール | エラーメッセージ |
|---------|--------|---------------|
| VR-030 | 開始日 ≤ 終了日 | 終了日は開始日以降にしてください |
| VR-031 | 被代理者 ≠ 代理者 | 自分自身を代理者に設定できません |
| VR-032 | 同一被代理者の有効期間重複不可 | 指定期間に既に代理設定が存在します |

## 2. 承認ルート判定ルール

### 2.1 固定2段階承認ルート

| ルールID | ルール | 説明 |
|---------|--------|------|
| AR-001 | 第1段階: 直属上長 | 申請者のsupervisor_idで特定 |
| AR-002 | 第2段階: 部門長 | 申請者のdepartment.manager_idで特定 |
| AR-003 | 直属上長 = 部門長の場合 | 1段階のみ（部門長承認で完了） |
| AR-004 | 代理承認者の判定 | 承認者に有効な代理設定がある場合、代理者も承認可能 |
| AR-005 | 承認ルートの有効期間 | ApprovalRouteのvalid_from/valid_toで判定 |

### 2.2 承認ルート判定フロー
```
1. 申請者の所属部門を取得
2. 申請者の直属上長を取得
3. 部門の部門長を取得
4. 直属上長 = 部門長 の場合:
   → 1段階承認（部門長のみ）
5. 直属上長 ≠ 部門長 の場合:
   → 2段階承認（直属上長 → 部門長）
6. 各段階の承認者について代理設定を確認
```

## 3. ステータス遷移ルール

| ルールID | ルール | 説明 |
|---------|--------|------|
| SR-001 | DRAFT → SUBMITTED | 申請者のみ実行可能。バリデーション通過が必要 |
| SR-002 | DRAFT → CANCELLED | 申請者のみ実行可能 |
| SR-003 | SUBMITTED → PENDING_MANAGER | 直属上長（または代理承認者）のみ実行可能 |
| SR-004 | SUBMITTED → REJECTED | 直属上長（または代理承認者）のみ実行可能 |
| SR-005 | SUBMITTED → CANCELLED | 申請者のみ実行可能 |
| SR-006 | PENDING_MANAGER → APPROVED | 部門長（または代理承認者）のみ実行可能 |
| SR-007 | PENDING_MANAGER → REJECTED | 部門長（または代理承認者）のみ実行可能 |
| SR-008 | REJECTED → RESUBMITTED | 申請者のみ実行可能。バリデーション通過が必要 |
| SR-009 | RESUBMITTED → PENDING_MANAGER | 直属上長（または代理承認者）のみ実行可能 |
| SR-010 | RESUBMITTED → REJECTED | 直属上長（または代理承認者）のみ実行可能 |
| SR-011 | RESUBMITTED → CANCELLED | 申請者のみ実行可能 |
| SR-012 | APPROVED, CANCELLED は最終ステータス | これ以上の遷移不可 |

## 4. 権限制御ルール

### 4.1 ロール定義

| ロール | 説明 |
|--------|------|
| EMPLOYEE | 一般社員 |
| MANAGER | 管理職（承認権限あり） |
| ACCOUNTING | 経理担当 |
| ADMIN | システム管理者 |

### 4.2 機能別権限マトリクス

| 機能 | EMPLOYEE | MANAGER | ACCOUNTING | ADMIN |
|------|----------|---------|------------|-------|
| 経費精算申請（自分） | ○ | ○ | ○ | ○ |
| 代理申請 | △（設定時） | △（設定時） | △（設定時） | ○ |
| 承認・差戻し | - | ○ | - | - |
| 代理承認 | - | △（設定時） | - | - |
| 申請照会（自分） | ○ | ○ | ○ | ○ |
| 申請照会（部下） | - | ○ | - | ○ |
| 申請照会（全件） | - | - | ○ | ○ |
| ダッシュボード | - | ○（部門） | ○（全体） | ○（全体） |
| ユーザー管理 | - | - | - | ○ |
| 承認ルート設定 | - | - | - | ○ |
| 代理設定管理 | - | - | - | ○ |
| 監査ログ閲覧 | - | - | ○ | ○ |

凡例: ○ = 許可、△ = 条件付き許可、- = 不許可

### 4.3 データアクセス制御

| ルールID | ルール | 説明 |
|---------|--------|------|
| AC-001 | 申請データの参照 | 申請者本人、承認者、代理承認者、経理担当、管理者のみ |
| AC-002 | 申請データの編集 | 申請者本人のみ（DRAFT, REJECTED ステータス時のみ） |
| AC-003 | 承認操作 | 該当段階の承認者または代理承認者のみ |
| AC-004 | ユーザー管理 | ADMINロールのみ |
| AC-005 | 監査ログ参照 | ACCOUNTING, ADMINロールのみ |

## 5. データ保管・アーカイブルール

| ルールID | ルール | 説明 |
|---------|--------|------|
| DR-001 | 申請データ保管期間 | 3年間 |
| DR-002 | 添付ファイル保管期間 | 申請データと同一（3年間） |
| DR-003 | 監査ログ保管期間 | 3年間 |
| DR-004 | アーカイブ処理 | 保管期間経過後、S3 Glacier等へアーカイブ |
| DR-005 | 削除処理 | アーカイブ後さらに1年経過で完全削除 |
| DR-006 | バッチ実行ログ | 1年間保管 |

## 6. 滞留申請判定ルール

| ルールID | ルール | 説明 |
|---------|--------|------|
| ST-001 | 滞留判定基準 | 承認待ち状態（SUBMITTED, PENDING_MANAGER, RESUBMITTED）が3営業日以上経過 |
| ST-002 | 営業日計算 | 土日・祝日を除く（祝日マスタは将来対応、当面は土日のみ除外） |
| ST-003 | ダッシュボード表示 | 滞留申請件数と一覧を表示 |
