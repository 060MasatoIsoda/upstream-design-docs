# ビジネスロジックモデル — expense-mvp

## 1. 経費精算申請のライフサイクル

### 1.1 ステータス一覧

| ステータス | 説明 |
|-----------|------|
| DRAFT | 下書き（未提出） |
| SUBMITTED | 申請済み（承認待ち: 直属上長） |
| PENDING_MANAGER | 承認待ち（部門長） |
| APPROVED | 承認済み |
| REJECTED | 差戻し |
| CANCELLED | 取消し |
| RESUBMITTED | 再申請済み（差戻し後の再提出、承認待ち: 直属上長） |

### 1.2 ステータス遷移図

```
                    +-------+
                    | DRAFT |
                    +-------+
                        |
                   [申請提出]
                        |
                        v
                  +-----------+
            +---->| SUBMITTED |<----+
            |     +-----------+     |
            |           |           |
            |      [上長承認]       |
            |           |           |
            |           v           |
            |   +----------------+  |
            |   |PENDING_MANAGER |  |
            |   +----------------+  |
            |           |           |
            |    [部門長承認]       |
            |           |           |
            |           v           |
            |     +----------+      |
            |     | APPROVED |      |
            |     +----------+      |
            |                       |
       [再申請]                [再申請]
            |                       |
            |     +----------+      |
            +-----| REJECTED |------+
                  +----------+
                  (上長差戻し or 部門長差戻し)

    ※ DRAFT, SUBMITTED, RESUBMITTED から CANCELLED への遷移が可能
    ※ RESUBMITTED は SUBMITTED と同じ承認フローに入る
```

### 1.3 ステータス遷移ルール

| 現在のステータス | アクション | 次のステータス | 実行者 |
|----------------|-----------|--------------|--------|
| DRAFT | 申請提出 | SUBMITTED | 申請者/代理申請者 |
| DRAFT | 取消し | CANCELLED | 申請者 |
| SUBMITTED | 上長承認 | PENDING_MANAGER | 直属上長/代理承認者 |
| SUBMITTED | 上長差戻し | REJECTED | 直属上長/代理承認者 |
| SUBMITTED | 取消し | CANCELLED | 申請者 |
| PENDING_MANAGER | 部門長承認 | APPROVED | 部門長/代理承認者 |
| PENDING_MANAGER | 部門長差戻し | REJECTED | 部門長/代理承認者 |
| REJECTED | 再申請 | RESUBMITTED | 申請者 |
| RESUBMITTED | 上長承認 | PENDING_MANAGER | 直属上長/代理承認者 |
| RESUBMITTED | 上長差戻し | REJECTED | 直属上長/代理承認者 |
| RESUBMITTED | 取消し | CANCELLED | 申請者 |

## 2. 承認ワークフロープロセスフロー

### 2.1 申請提出フロー
```
1. 申請者が経費精算申請を起票（または下書きから編集）
2. 入力バリデーション実行
   - 必須項目チェック
   - 金額上限チェック（1申請100万円以下）
   - 添付ファイルサイズチェック
3. 承認ルート判定
   - 申請者の所属部門を取得
   - 直属上長を取得
   - 部門長を取得
4. 承認レコード生成（2段階分）
5. ステータスを SUBMITTED に更新
6. 直属上長へ承認依頼メール送信
7. 監査ログ記録
```

### 2.2 承認フロー
```
1. 承認者が承認待ち一覧から申請を選択
2. 申請内容・添付ファイルを確認
3-A. 承認の場合:
   a. 承認レコードを更新（承認日時・承認者記録）
   b. 次段階の有無を判定
      - 次段階あり（上長承認後）: ステータスを PENDING_MANAGER に更新、部門長へ通知
      - 次段階なし（部門長承認後）: ステータスを APPROVED に更新、申請者へ完了通知
   c. 監査ログ記録
3-B. 差戻しの場合:
   a. 差戻しコメントを記録
   b. ステータスを REJECTED に更新
   c. 申請者へ差戻し通知メール送信
   d. 監査ログ記録
```

### 2.3 再申請フロー
```
1. 申請者が差戻し済み申請を開く
2. 申請内容を修正
3. 入力バリデーション実行
4. 承認ルートを再判定（組織変更に対応）
5. 新しい承認レコード生成
6. ステータスを RESUBMITTED に更新
7. 直属上長へ承認依頼メール送信
8. 監査ログ記録（修正前後の差分を記録）
```

## 3. 代理申請・代理承認ロジック

### 3.1 代理申請
- 代理申請者は、被代理者（本来の申請者）を指定して申請を起票する
- 申請データには「申請者」と「代理申請者」の両方を記録する
- 承認ルートは「申請者」の所属部門に基づいて判定する
- 代理申請の権限はシステム管理者が事前に設定する

### 3.2 代理承認
- 代理承認者は、被代理者（本来の承認者）に代わって承認・差戻しを行う
- 代理設定には期間（開始日・終了日）を持たせる
- 期間内のみ代理承認が有効となる
- 承認レコードには「承認者」と「代理承認者」の両方を記録する
- 代理承認者の承認待ち一覧には、被代理者宛の申請も表示する

### 3.3 代理設定の管理
- システム管理者が代理設定を登録・編集・削除する
- 設定項目: 被代理者、代理者、開始日、終了日
- 1人の被代理者に対して同時に有効な代理者は1人のみ
- 期間が重複する代理設定は登録不可

## 4. バッチ処理ロジック

### 4.1 人事マスタ取込
```
1. S3バケットからCSVファイルを取得
2. CSVフォーマットバリデーション
3. 各レコードについて:
   a. 社員番号で既存ユーザーを検索
   b. 存在する場合: 差分を検出し更新
   c. 存在しない場合: 新規ユーザーとして登録
   d. CSVに存在しない既存ユーザー: 無効化フラグを立てる
4. 処理結果サマリーをログ記録（新規/更新/無効化/エラー件数）
5. エラーレコードをエラーファイルとして出力
```

### 4.2 組織マスタ取込
```
1. S3バケットからCSVファイルを取得
2. CSVフォーマットバリデーション
3. 各レコードについて:
   a. 組織コードで既存組織を検索
   b. 存在する場合: 差分を検出し更新（組織名、親組織、部門長）
   c. 存在しない場合: 新規組織として登録
4. 承認ルートの整合性チェック（部門長変更時）
5. 処理結果サマリーをログ記録
6. エラーレコードをエラーファイルとして出力
```

### 4.3 会計仕訳データ出力
```
1. 対象データ抽出: ステータスが APPROVED かつ未出力の経費精算申請
2. 各申請の明細行について:
   a. 経費種別から勘定科目を取得（マスタ参照）
   b. 仕訳データに変換（借方・貸方・金額・摘要・部門コード）
3. CSVファイルとして出力
4. 出力済みフラグを更新
5. 処理結果サマリーをログ記録
```

## 5. 通知トリガーとテンプレート

### 5.1 通知トリガー一覧

| トリガー | 送信先 | テンプレート |
|---------|--------|------------|
| 申請提出 | 直属上長（または代理承認者） | 承認依頼通知 |
| 上長承認 | 部門長（または代理承認者） | 承認依頼通知 |
| 最終承認 | 申請者 | 承認完了通知 |
| 差戻し | 申請者 | 差戻し通知 |
| 再申請 | 直属上長（または代理承認者） | 承認依頼通知（再申請） |

### 5.2 メールテンプレート構成

各テンプレートに含む情報:
- 件名（申請種別・申請番号を含む）
- 申請者名
- 申請日
- 合計金額
- 申請概要（経費種別・件数）
- 差戻しコメント（差戻し通知の場合）
- 申請詳細画面へのリンク

### 5.3 通知言語
- ユーザーの表示言語設定に基づいて日本語または英語でメール送信
